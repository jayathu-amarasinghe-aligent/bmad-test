## Directory Purpose

### /src (Application Source)
- **app/**: Next.js 16 App Router pages with file-based routing
- **components/**: Reusable React components (Server & Client)
- **stores/**: Zustand state management stores
- **lib/**: Utility functions, metadata factories, sync scripts
- **types/**: TypeScript type definitions
- **data/**: Static JSON data generated by GitHub Actions

### /.github (CI/CD Workflows)
- **sync-images.yml**: Daily automated image sync (2 AM UTC)
- **ci.yml**: Quality gates (ESLint, TypeScript, Playwright tests)

### /e2e (End-to-End Tests)
- Playwright test specs for critical user journeys
- Homepage, gallery navigation, lightbox, social conversions

### /public (Static Assets)
- Favicon, default Open Graph images
- Robots.txt generated by app/robots.ts

### Configuration Files (Root)
- Next.js, TypeScript, ESLint, Tailwind, Playwright configs
- Package.json with all dependencies
- Environment variables (.env.local, .env.example)

## Key Boundaries

### Server Components (Green)
- All pages in `/src/app` by default
- No client-side JavaScript unless needed

### Client Components (Yellow)
- Marked with `'use client'` directive
- Interactive components only (TimeAdaptiveHero, MasonryGallery, etc.)

### State Management (Red)
- Zustand stores in `/src/stores`
- Client components only

### Build Process
- GitHub Actions commits → Vercel build → Static export
- Zero manual intervention required
